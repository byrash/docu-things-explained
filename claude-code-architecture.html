DFD-1 Level-1 Data Flow Diagram

Clean swim-lane layout. Each lane is a horizontal band. Data flows left-to-right through processes, with orthogonal routing — no overlapping lines.

Lane A — User Input & Prompt Assembly
Lane B — API Call & Streaming
Lane C — Tool Execution (when tool_use present)
Lane D — Persistence & Memory
User / Prompt Data

How to read: Each lane is a separate phase. Data flows left → right within a lane. The P3 queryLoop() process appears in every lane because it orchestrates the entire cycle: it sends messages right (Lane B), dispatches tools (Lane C), persists data (Lane D), and loops back when tool results arrive.
